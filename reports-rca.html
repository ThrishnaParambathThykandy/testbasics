Requirement Traceability Matrix (RTM)

RTM is a document that maps requirements to corresponding test cases to ensure complete test coverage.

It helps identify missing test cases and ensures all requirements are validated.

Types of Risks

  • Project Risk: Schedule, cost, resource issues
  • Product Risk: Quality, performance, security defects
  • Business Risk: Customer dissatisfaction, compliance issues

Entry Criteria

Entry criteria define the conditions that must be met before testing can begin.

Example: Approved requirements, test environment ready.

Exit Criteria

Exit criteria define the conditions that must be met to stop testing.

Example: No critical defects open, test cases executed.

Root Cause Analysis (RCA)

RCA is a technique used to identify the underlying cause of defects so that similar issues do not occur again.

Test Metrics

  • Test case execution percentage
  • Pass / Fail rate
  • Defect density
  • Defect leakage

Project Summary Report (Test Closure Report)

A Project Summary Report provides an overview of the testing activities performed during a release.

It includes test execution summary, defect status, risks, lessons learned, and sign-off.
