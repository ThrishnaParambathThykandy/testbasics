What is API Testing?

API testing verifies the functionality, reliability, and performance of backend services without using the user interface.

Why API Testing?

  • Faster than UI testing
  • Detects defects early
  • Validates business logic
  • Independent of UI changes

What is Swagger?

Swagger is an API documentation tool that helps testers understand endpoints, request parameters, and response formats.

Common HTTP Methods

  • GET: Retrieve data
  • POST: Create new data
  • PUT: Update existing data
  • DELETE: Remove data

HTTP Status Codes

  • 200: OK
  • 201: Created
  • 400: Bad Request
  • 401: Unauthorized
  • 404: Not Found
  • 500: Internal Server Error

Basic API Test Scenarios

  • Validate response status code
  • Validate response body
  • Validate error handling
  • Check response time

What is SQL Testing?

SQL testing validates whether data stored in the database is correct after application actions.

SELECT Statement

Used to retrieve data from a database table.

Example:

SELECT * FROM users;

WHERE Clause

Filters records based on conditions.

Example:

SELECT * FROM users WHERE status = 'ACTIVE';

SQL JOINS

Joins combine data from multiple tables.

  • INNER JOIN
  • LEFT JOIN
  • RIGHT JOIN

Nested Queries (Subqueries)

A query inside another query.

Example:

SELECT name FROM users WHERE id IN (SELECT user_id FROM orders);
